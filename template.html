Android developers

There are two IDEs option for android developers. All those are cross-platforms.

  • ADT (android developer tools).
    • Refactoring of eclipse + some bundles.
    • Clearly the most featured and used nowdays.
    • An alternative is installing eclipse with the adt plugin.
  • Android Studio: based on intellij IDEA.
    • First presented at google IO 2013.
    • Clearly where the google developer tools team effort is.
    • More than just the IDE: switch of build system, more integration.
    • Still in alpha (0.5.x).

Installing different sdk, emulators, documentation and so on…

Android developers

Proposal

  • Not a lot is needed to make the developer life setup way better. We can only focus on the setup experience this cycle.
  • We can skip the first whole part of the process, creating the ubuntu developer tool center.
    • Proposing for download and setup where the user wants (with sane defaults) adt or android studio (with the alpha flag). Always the latest version.
    • Installing the openjdk vm and setup it up by default.
    • Setting up the needed quirks if on 64bits
    • Add a launcher icon and try to setup sdk/vms pre-launchers.
    • Upgrade mecanism and first setup download with SDK download is let to the great auto-update from those tools.
  • It can be a simple command line tool, with a GUI if needed, letting you choose between options.
  • Can be easily extended to include other platforms, like dart… maybe unity3d…
  • The SDK is part of the download, but it stays on google server…

  • If we are downloading from the trusted source, we need to handle the ssl check and integrity through md5sum.
